# Lab_2_Stream
## Опис
Зробив Марченко Артем ІА-34.Проект містить реалізацію роботи з Stream API
## Завдання
1. Визначити C4 як остачу від ділення номеру залікової книжки студента/студентки на 4: C4=11=3 mod 4.

| C4 | Домена область | Поле А | Поле Б | Поле В | Поле Г |
|----|---------------------------|-----|-------------------------------------------------------|------------|------------|
| 3  | Хтонічні істоти (нечисть). Обов’язкова інформація: ім’я, вид істоти, дата першої згадки в (можливо окультній) літературі, сила атаки |вид    |кількість років з моменту першої згадки в літературі                                                 | вид істоти  | сила атаки        |

2. Створити нескінченний генератор в стрім, який створює об’єкти із доменної області з випадковими (але бажано реалістичними) параметрами

3.Згенерувати список з 500 об’єктів із генератора, відкинувши перші N (змінний параметр) об’єктів за певним значенням із Поля A. (результуючий список має містити 500 елементів, для реалізації написати Gatherer)

4.Відфільтрувати отриманий список за діапазоном значень Параметра Б та згрупувати значення за Полем В.

5.Дослідити список із п.4: Створити клас для статистичних даних, та зібрати у нього (за допомогою свого Collector) елементи зі списку. Вивести  на екран наступні значення: мінімальне і максимальне значення поля Г, середнє значення поля Г та стандартне відхилення. 

6.Дослідити значення поля Г в отриманому списку: визначити  міжквартильний розмах та ідентифікувати викиди в отриманому списку. Згрупувати список по належності значення поля Г до викидів та підрахувати кількість значень у кожній категорії.
